# 报表开发向导

## 1. 模块说明

`报表中心` 提供统一的报表目录、在线配置、数据预览、导出和打印能力。

核心入口：

- 页面：`Known/Pages/SysReportList.razor`
- 服务：`Known/Services/ReportService.cs`
- 实体：`Known/Entities/SysReport.cs`
- 设计表单：`Known/Pages/ReportForm.razor`

## 2. 报表定义模型

`SysReport` 用于保存报表元数据，主要字段：

- `Code`：报表代码
- `Name`：报表名称
- `BizType`：业务类型，用于不同模块隔离报表目录
- `Category`：分组
- `Icon`：图标
- `SourceType`：数据源类型，支持 `Table` / `Sql`
- `Source`：数据源内容
  - `Table`：填写表名
  - `Sql`：填写完整查询 SQL
- `Fields`：报表字段定义列表
- `IsPaging`：是否分页

`ReportFieldInfo` 用于描述字段显示与查询行为，常用字段：

- `Id`：字段ID，必须与查询结果字段名一致
- `Name`：显示名称
- `Expression`：字段表达式
- `Type`：字段类型
- `Category`：代码表类别
- `IsQuery`：是否作为查询条件
- `IsVisible`：是否显示
- `DefaultSort`：默认排序（`asc` / `desc`）

## 3. 使用方式

### 3.1 新建报表

进入 `系统管理 -> 报表中心`：

1. 点击 `新建报表`
2. 填写基本信息
3. 配置数据源
4. 配置字段列表
5. 点击 `在线预览`
6. 保存后即可在左侧报表目录中查看

### 3.2 数据源配置

#### Table 模式

- `SourceType = Table`
- `Source = 实际表名`
- 点击 `加载表字段` 自动生成字段定义

适合：单表、快速报表、低代码开发。

#### Sql 模式

- `SourceType = Sql`
- `Source = 完整 SQL`
- 需要保证查询列别名与 `字段ID` 一致

示例：

`select Code as Code, Name as Name, CreateTime as CreateTime from SysUser`

适合：多表关联、统计报表、复杂业务报表。

## 4. 字段开发建议

### 4.1 Table 模式推荐

优先通过 `加载表字段` 自动生成，再按需调整：

- 名称
- 类型
- 是否查询
- 是否显示
- 默认排序
- 代码表类别

### 4.2 Sql 模式推荐

先写可运行 SQL，再配置字段：

- `Id` 对应 SQL 别名
- `Expression` 可为空
- 字段显示顺序按配置列表顺序决定

## 5. 扩展业务报表

`SysReportList` 提供可覆写的 `BizType`：

- 默认页面显示全局报表
- 业务模块可继承 `SysReportList`
- 覆写 `BizType` 后，仅查询对应业务报表

示例：

- 订单模块：`protected override string BizType => "Order";`
- 生产模块：`protected override string BizType => "Produce";`

## 6. 导出与打印

当前支持：

- 表格导出 Excel
- 当前结果打印

说明：

- 导出基于当前字段配置生成列
- 打印基于当前查询结果生成表格内容

## 7. 在线开发建议

如果希望在线开发更便捷，建议按以下顺序：

1. 优先选择 `Table` 模式
2. 自动加载表字段
3. 标记可查询字段
4. 设置默认排序
5. 通过 `在线预览` 校验结果
6. 保存发布

## 8. 后续可增强方向

当前报表中心已具备在线配置基础，后续可继续扩展：

- 字段聚合统计

- SQL 参数模板
- 拖拽式字段布局
- 报表权限控制
- 报表模板分享与复制
